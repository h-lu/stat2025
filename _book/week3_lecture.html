  • summarise() (或 summarize()): 数据汇总

    • 通常与 group_by() 结合使用，对每个分组进行汇总计算。
    • 创建包含汇总统计量的新数据框。
    • 常用的汇总函数：mean(), median(), sd(), var(), min(), max(), n() (计算行数/观测数), n_distinct() (计算唯一值数量)。
    # 计算所有学生的平均年龄和最高 GPA (没有 group_by)
students_data %>%
  summarise(
    Avg_Age = mean(Age),
    Max_GPA = max(GPA)
  )
    #> # A tibble: 1 × 2
#>   Avg_Age Max_GPA
#>     <dbl>   <dbl>
#> 1      21     3.9
    # 计算每个专业的学生人数和平均 GPA
students_data %>%
  group_by(Major) %>%
  summarise(
    Num_Students = n(), # n() 计算当前分组的行数
    Avg_GPA = mean(GPA, na.rm = TRUE) # 好习惯：处理可能的NA
  )
    #> # A tibble: 3 × 3
#>   Major            Num_Students Avg_GPA
#>   <chr>                   <int>   <dbl>
#> 1 Computer Science            1     3.5
#> 2 Mathematics                 1     3.9
#> 3 Statistics                  2     3.7
    # 计算每个学生的考试次数和平均分
exam_scores %>%
  group_by(StudentID) %>%
  summarise(
    Num_Exams = n(),
    Avg_Score = mean(Score),
    Min_Score = min(Score),
    Max_Score = max(Score)
  )
    #> # A tibble: 3 × 5
#>   StudentID Num_Exams Avg_Score Min_Score Max_Score
#>       <dbl>     <int>     <dbl>     <dbl>     <dbl>
#> 1       101         3      89.3        85        95
#> 2       102         2      91          90        92
#> 3       103         2      77.5        75        80
    # 计算每门考试的参加人数和最高分
exam_scores %>%
  group_by(Exam) %>%
  summarise(
    Num_Participants = n_distinct(StudentID), # 统计不重复的学生ID数
    Highest_Score = max(Score)
  )
    #> # A tibble: 3 × 3
#>   Exam    Num_Participants Highest_Score
#>   <chr>              <int>         <dbl>
#> 1 Final                  3            92
#> 2 Midterm                3            90
#> 3 Quiz1                  1            95
    ungroup()

    使用 group_by() 后，数据会保持分组状态。如果后续操作希望在整个数据集上进行，而不是分组进行，需要使用 ungroup() 取消分组。

  ungroup() %>% # 取消分组
  mutate(Overall_Avg_GPA = mean(GPA)) # 在全体数据上计算总平均GPA

#>   Name      Age Major              GPA Major_Avg_GPA Overall_Avg_GPA

2. 整洁数据 (Tidy Data) 的理念

“整洁数据” 是由 Hadley Wickham 提出的数据组织标准，旨在简化数据分析流程。遵循整洁数据原则的数据更易于使用 tidyverse 工具进行处理和可视化。

  • 三条核心原则:
    1. 每个变量构成一列 (Each variable forms a column)。
    2. 每个观测构成一行 (Each observation forms a row)。
    3. 每种类型的观测单元构成一个表格 (Each type of observational unit forms a table)。
  • 为何重要？
    • tidyverse 中的函数（dplyr, ggplot2 等）都设计为处理整洁数据。
    • 使数据结构标准化，便于理解和协作。
    • 简化数据清理和转换步骤。
  • 不整洁数据的常见形式:
    • 列名是变量值，而不是变量名 (例如，年份作为列名)。
    • 一个单元格中存储了多个值。
    • 变量同时存储在行和列中。

3. tidyr: 让数据更整洁

tidyr 包提供了将数据在”宽”格式和”长”格式之间转换的函数，帮助我们将数据整理成”整洁”的形式。

  • 宽数据 (Wide Data): 通常人类阅读更友好，但不利于计算。变量值可能作为列名。

  • 长数据 (Long Data): 通常更符合整洁数据原则，便于 tidyverse 处理。每个观测的关键信息都在行内，变量名在列中。

  • 核心函数:

    • pivot_longer(): 将宽数据变长。将多个列”融合”成键-值对的两列。
    • pivot_wider(): 将长数据变宽。根据键-值对将数据”展开”到多个列。

  • pivot_longer() 示例: 假设我们有如下”宽”格式数据，记录了不同季度销售额：

    sales_wide <- tibble(
  Product = c("A", "B"),
  Qtr1 = c(100, 150),
  Qtr2 = c(110, 160),
  Qtr3 = c(120, 170),
  Qtr4 = c(130, 180)
  )

  print(sales_wide)
    #> # A tibble: 2 × 5
#>   Product  Qtr1  Qtr2  Qtr3  Qtr4
#>   <chr>   <dbl> <dbl> <dbl> <dbl>
#> 1 A         100   110   120   130
#> 2 B         150   160   170   180

    这里的列名 Qtr1 到 Qtr4 实际上是”季度”这个变量的值。我们希望将其转换为整洁的长格式：

    sales_long <- sales_wide %>%
  pivot_longer(
    cols = Qtr1:Qtr4, # 指定哪些列需要被转换 (这里是 Qtr1 到 Qtr4) 
    names_to = "Quarter",  # 新列名，用于存储原来的列名 (Qtr1, Qtr2...)
    values_to = "Sales"    # 新列名，用于存储原来的单元格值
  )

print(sales_long)
    #> # A tibble: 8 × 3
#>   Product Quarter Sales
#>   <chr>   <chr>   <dbl>
#> 1 A       Qtr1      100
#> 2 A       Qtr2      110
#> 3 A       Qtr3      120
#> 4 A       Qtr4      130
#> 5 B       Qtr1      150
#> 6 B       Qtr2      160
#> 7 B       Qtr3      170
#> 8 B       Qtr4      180

    现在，每个观测（某个产品在某个季度的销售额）都独占一行，变量（产品、季度、销售额）各自成列。

  • pivot_wider() 示例: 将刚才的长数据 sales_long 转换回宽数据：

    sales_regained_wide <- sales_long %>%
  pivot_wider(
    names_from = Quarter, # 指定包含新列名的列 (Quarter)
    values_from = Sales   # 指定包含新单元格值的列 (Sales)
    )

print(sales_regained_wide)
    #> # A tibble: 2 × 5
#>   Product  Qtr1  Qtr2  Qtr3  Qtr4
#>   <chr>   <dbl> <dbl> <dbl> <dbl>
#> 1 A         100   110   120   130
#> 2 B         150   160   170   180

4. 处理缺失值 (NA)

缺失值 (NA, Not Available) 在真实数据中非常常见。dplyr 和 tidyr 以及 R 基础函数提供了处理它们的方法。

  • 识别缺失值: is.na() 函数返回一个逻辑向量，TRUE 表示对应位置是 NA。

    x <- c(1, 2, NA, 4, NA)
is.na(x)
    #> [1] FALSE FALSE  TRUE FALSE  TRUE
    # 在数据框中使用 (例如，检查 exam_scores 中是否有 NA)
is.na(exam_scores) # 返回一个与数据框同样大小的逻辑矩阵
    #>      StudentID  Exam Score
#> [1,]     FALSE FALSE FALSE
#> [2,]     FALSE FALSE FALSE
#> [3,]     FALSE FALSE FALSE
#> [4,]     FALSE FALSE FALSE
#> [5,]     FALSE FALSE FALSE
#> [6,]     FALSE FALSE FALSE
#> [7,]     FALSE FALSE FALSE
    any(is.na(exam_scores)) # 检查整个数据框是否有任何 NA
    #> [1] FALSE
    colSums(is.na(exam_scores)) # 计算每列的 NA 数量
    #> StudentID      Exam     Score 
#>         0         0         0

  • 处理策略:

    1. 删除:

      • na.omit() (基础 R): 删除任何列包含 NA 的整行。简单粗暴，可能丢失过多信息。
      • drop_na() (tidyr): 功能类似 na.omit(), 但可以指定只检查某些列的 NA。
    scores_na_df <- tibble(ID = 1:6, Score = c(85, 92, NA, 88, 95, 72))
print(scores_na_df)
    #> # A tibble: 6 × 2
#>      ID Score
#>   <int> <dbl>
#> 1     1    85
#> 2     2    92
#> 3     3    NA
#> 4     4    88
#> 5     5    95
#> 6     6    72
    na.omit(scores_na_df) # 删除包含 NA 的行
    #> # A tibble: 5 × 2
#>      ID Score
#>   <int> <dbl>
#> 1     1    85
#> 2     2    92
#> 3     4    88
#> 4     5    95
#> 5     6    72
    library(tidyr)
scores_na_df %>% drop_na() # 同上
    #> # A tibble: 5 × 2
#>      ID Score
#>   <int> <dbl>
#> 1     1    85
#> 2     2    92
#> 3     4    88
#> 4     5    95
#> 5     6    72
    # 假设有另一列，我们只想删除 Score 为 NA 的行
scores_na_df2 <- tibble(
  ID = 1:6,
  Score = c(85, 92, NA, 88, 95, 72),
  Group = c("A", "B", "A", NA, "B", "A")
  )
print(scores_na_df2)
    #> # A tibble: 6 × 3
#>      ID Score Group
#>   <int> <dbl> <chr>
#> 1     1    85 A    
#> 2     2    92 B    
#> 3     3    NA A    
#> 4     4    88 <NA> 
#> 5     5    95 B    
#> 6     6    72 A
    scores_na_df2 %>% drop_na(Score) # 只检查 Score 列的 NA
    #> # A tibble: 5 × 3
#>      ID Score Group
#>   <int> <dbl> <chr>
#> 1     1    85 A    
#> 2     2    92 B    
#> 3     4    88 <NA> 
#> 4     5    95 B    
#> 5     6    72 A
    1. 填充/插补 (Imputation): 用某个值（如均值、中位数、众数或模型预测值）替换 NA。这更复杂，需要根据具体情况选择合适的方法（后续课程可能涉及）。
    • replace_na() (tidyr): 用指定值替换 NA。
    scores_na_df %>%
  mutate(Score_filled = replace_na(Score, mean(Score, na.rm = TRUE))) # 用均值填充
    #> # A tibble: 6 × 3
#>      ID Score Score_filled
#>   <int> <dbl>        <dbl>
#> 1     1    85         85  
#> 2     2    92         92  
#> 3     3    NA         86.4
#> 4     4    88         88  
#> 5     5    95         95  
#> 6     6    72         72
    1. 在计算中忽略: 许多函数（如 mean(), sum(), sd() 等）有 na.rm = TRUE 参数。
    mean(scores_na_df$Score) # NA
    #> [1] NA
    mean(scores_na_df$Score, na.rm = TRUE) # 正确计算
    #> [1] 86.4
处理 NA 的重要性

不恰当处理 NA 会导致分析结果偏差甚至错误。选择哪种策略取决于数据的性质、缺失的原因以及分析的目标。删除是最简单的方法，但只有在 NA 占比很小或确定该观测无效时才推荐。

5. 管道 (%>%) 连接多步操作

管道使得复杂的数据处理流程清晰易读。

  • 示例： 从 exam_scores 数据中，找出每个学生（StudentID）的最高分（Max_Score），并按最高分降序排列，最后只显示 StudentID 和 Max_Score。

    exam_scores %>%
  group_by(StudentID) %>% # 按学生分组
  summarise(Max_Score = max(Score)) %>% # 计算每个学生的最高分
  arrange(desc(Max_Score)) %>% # 按最高分降序排列
  ungroup() # 最好取消分组（虽然这里影响不大）
    #> # A tibble: 3 × 2
#>   StudentID Max_Score
#>       <dbl>     <dbl>
#> 1       101        95
#> 2       102        92
#> 3       103        80

  • 对比不使用管道的代码：

    grouped_data <- group_by(exam_scores, StudentID)
summary_data <- summarise(grouped_data, Max_Score = max(Score))
arranged_data <- arrange(summary_data, desc(Max_Score))
ungroup(arranged_data)
    #> # A tibble: 3 × 2
#>   StudentID Max_Score
#>       <dbl>     <dbl>
#> 1       101        95
#> 2       102        92
#> 3       103        80

管道显然更简洁、更符合思维流程。
