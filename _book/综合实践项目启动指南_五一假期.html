综合实践项目启动详细指南 (五一假期任务)

大家好！

综合实践项目是我们应用所学知识、深入探索数据分析过程的重要机会。这个”五一”假期是启动项目、奠定坚实基础的绝佳时机。本指南将详细分解假期内需要完成的核心任务，确保你能有效地开始你的数据分析之旅。

假期核心目标： 完成项目选题、数据准备和初步探索性数据分析 (EDA)。

任务一：最终确定选题与研究问题

这是项目的起点和方向，至关重要。一个清晰、具体的问题将指导你后续所有的数据分析工作。

  1. 从兴趣出发，聚焦具体问题：
    • 回顾你初步构思的 1-2 个主题。哪个最让你感兴趣？哪个的数据看起来更容易获取和处理？
    • 将宽泛的主题转化为一个或两个可以明确回答的问题。避免过于宏大或模糊不清的问题。
    • 检查问题是否”SMART”：
      • Specific (具体的): 问题明确指向特定的现象或关系吗？
      • Measurable (可衡量的): 你能用数据来量化和衡量问题中的关键概念吗？
      • Achievable (可实现的): 你拥有的数据和当前掌握的技能足以回答这个问题吗？
      • Relevant (相关的): 这个问题与你的兴趣、专业或课程内容相关吗？
      • Time-bound (有时限的): (在项目整体框架下) 这个问题能在课程时间内完成分析吗？
  2. 问题示例 (更通用):
    • 好问题 (具体、可衡量):
      • 描述性/比较性: “不同产品类别 (category) 的平均销售额 (sales) 是否存在差异？哪个类别平均销售额最高？” (适用于 EDA 的分组箱线图或后续的 ANOVA)
      • 描述性/比较性: “来自不同地区 (region) 的用户在网站停留时间 (session_duration) 上有何不同？” (适用于 EDA 的分组箱线图)
      • 关系探索: “广告支出 (ad_spend) 与网站访问量 (visits) 之间是否存在关联？它们的关系是怎样的？” (适用于 EDA 的散点图或后续的相关/简单回归分析)
      • 关系探索: “顾客年龄 (age) 与其购买金额 (purchase_amount) 之间有关系吗？” (适用于 EDA 的散点图)
      • 分类导向: “哪些用户特征（如 age, region, past_purchases）可以帮助区分高价值客户和普通客户？” (引向分类问题)
    • 待改进的问题 (过于宽泛):
      • “分析销售数据。” (分析什么？目标是什么？)
      • “研究用户行为。” (研究哪些行为？与什么相关？)
  3. 最终确认： 写下你最终确定的 1-2 个核心研究问题。确保你理解这些问题，并相信你的数据能够帮助回答它们。

任务二：确保数据到位：获取、导入与整理

数据是分析的基础。你需要确保你的”原材料”已经准备就绪。

  1. 获取数据：
    • 根据你的选题，下载所需的数据集。
    • 来源可能包括：老师提供的文件、Kaggle、UCI 机器学习库、天池、国家/地方政府数据开放平台、特定研究机构网站等。
    • 建议： 将下载的原始数据文件保存在你电脑上一个清晰、易于查找的位置。
  2. 创建项目结构 (推荐)：
    • 在你的电脑上为这个项目创建一个总文件夹。
    • 在总文件夹内，建议创建子文件夹，例如：
      • data/: 存放原始数据文件和清理后的数据文件。
      • scripts/ 或 code/: 存放你的 R 脚本文件 (.R 或 .qmd)。
      • plots/: (可选) 存放生成的图表文件。
      • reports/: (可选) 存放最终的报告文件。
    • 这样做有助于保持项目文件的整洁和有序。
  3. 导入数据到 R 环境：
    • 启动 RStudio，建议创建一个新的 R 项目 (File -> New Project…) 并将其设置在你创建的总文件夹中。这样可以更好地管理工作目录。
    • 使用 readr 包 (属于 tidyverse) 中的函数导入数据。最常用的是：
      • read_csv("path/to/your/data.csv"): 读取逗号分隔值文件 (.csv)。这是最常见的数据格式。
      • read_tsv("path/to/your/data.tsv"): 读取制表符分隔值文件 (.tsv)。
      • (如果需要读取 Excel 文件，可能需要 readxl 包)：readxl::read_excel("path/to/your/data.xlsx")。
    • 路径: "path/to/your/data.csv" 应替换为你数据文件的实际路径。如果 R 项目设置正确，且数据在 data/ 子文件夹下，路径通常是 "data/your_data.csv"。
    • 将读取的数据赋值给一个变量 (数据框)，例如：my_data <- read_csv("data/your_data.csv")。
  4. 检查导入是否成功：
    • head(my_data): 查看数据框的前几行。
    • tail(my_data): 查看数据框的后几行。
    • dim(my_data): 查看数据框的维度 (行数和列数)。
    • colnames(my_data): 查看所有列名。
    • 确保数据看起来符合预期，没有明显的读取错误。

任务三：完成初步探索性数据分析 (EDA)

EDA 的目的是深入了解你的数据，发现其内在规律、异常情况以及变量间的潜在联系，为后续的建模和分析提供依据。

  1. 数据整体检查 (使用 R 函数):
    • glimpse(my_data): 必用！快速概览数据结构，每列的名称、数据类型 (如 dbl 双精度浮点数, int 整数, chr 字符, fct 因子) 和前几行的值。特别关注数据类型是否符合预期 (例如，分类变量是否应为因子 fct？)。
    • summary(my_data): 提供数值变量的五数概括 (最小值、第一四分位数、中位数、均值、第三四分位数、最大值) 和缺失值计数 (NA’s)，以及字符/因子变量的频数统计。有助于发现异常值和数据分布的初步印象。
    • skimr::skim(my_data): (需要先安装和加载 skimr 包: install.packages("skimr"); library(skimr)) 提供一个非常详尽的概览，包括缺失值比例、数值变量的迷你直方图、字符变量的独特性等。强烈推荐使用！
    • 重点检查:
      • 缺失值 (NA): 哪些列有缺失？缺失比例高吗？ (这会影响后续分析)
      • 数据类型: 是否有需要转换类型的列？(例如，代表类别的数字应转为因子)
      • 数值范围: 数值变量的范围是否合理？是否存在明显的极端值或异常值？
      • 分类变量水平: 如果有分类变量 (因子)，其水平 (levels) 是否正确？例如，对于 Logistic 回归的因变量，确保代表”成功”或”目标事件”的水平是第二个。
  2. 单变量可视化 (ggplot2): 探索每个重要变量自身的分布特征。
    • 连续变量:
      • ggplot(my_data, aes(x = continuous_variable)) + geom_histogram(bins = 30): 直方图，观察分布形状 (对称、左偏、右偏)、峰值位置。bins 参数可以调整分组数量。
      • ggplot(my_data, aes(x = continuous_variable)) + geom_density(): 核密度图，平滑版的直方图，更清晰地展示分布形态。
      • ggplot(my_data, aes(y = continuous_variable)) + geom_boxplot(): 箱线图，展示中位数、四分位数范围 (IQR)、以及潜在的异常点。
    • 分类变量:
      • ggplot(my_data, aes(x = categorical_variable)) + geom_bar(): 条形图，展示每个类别的频数或计数。
  3. 变量间关系可视化 (ggplot2): 探索变量之间可能存在的联系。
    • 连续 vs. 连续:
      • ggplot(my_data, aes(x = var1, y = var2)) + geom_point(): 散点图，观察是否存在线性关系、非线性关系、聚类或异常点。
    • 分类 vs. 连续:
      • ggplot(my_data, aes(x = categorical_var, y = continuous_var)) + geom_boxplot(): 分组箱线图，比较不同类别下连续变量的分布、中位数、离散程度和异常值。非常常用！
      • ggplot(my_data, aes(x = categorical_var, y = continuous_var)) + geom_violin(): 分组小提琴图，是箱线图和密度图的结合，能更详细地展示分布形态。
      • (可选) ggplot(my_data, aes(x = continuous_var, fill = categorical_var)) + geom_density(alpha = 0.5): 分组密度图，比较不同类别下连续变量的分布曲线。
    • 分类 vs. 分类:
      • ggplot(my_data, aes(x = cat_var1, fill = cat_var2)) + geom_bar(position = "fill"): 填充条形图，展示在一个分类变量的每个类别中，另一个分类变量的相对比例。
      • ggplot(my_data, aes(x = cat_var1, fill = cat_var2)) + geom_bar(position = "dodge"): 并列条形图，比较不同类别组合的绝对频数。
      • my_data %>% count(cat_var1, cat_var2) %>% ggplot(aes(x = cat_var1, y = cat_var2, fill = n)) + geom_tile(): 热力图，用颜色深浅表示两个分类变量组合的频数。
  4. 记录你的发现！: 这是 EDA 最重要的一步。
    • 在代码中添加注释: 对你的代码进行解释，说明你为什么进行某个检查或绘图。
    • 记录关键观察: 在脚本的注释中，或者单独的笔记文件中，记下你从检查和图表中获得的主要信息。例如：
      • “变量 income 呈右偏分布，可能需要进行对数变换。”
      • “变量 age 与 purchase_amount 似乎存在正相关。”
      • “发现 location 列有 15% 的缺失值，需要后续处理。”
      • “不同 education_level 组的平均 salary 有明显差异，boxplot 显示…”
    • 保存重要图表: 将具有启发性的图表保存下来 (可以使用 ggsave("plots/my_plot.png"))，并给予有意义的文件名。

假期结束时你应该拥有：

  1. 一份明确的研究问题陈述。
  2. 一个包含你的 R 项目、数据文件和初步 R 脚本的项目文件夹。
  3. 一个可以成功运行的 R 脚本，该脚本能够：
    • 加载所需的库 (library(tidyverse), library(skimr) 等)。
    • 成功导入你的数据集。
    • 包含你进行数据整体检查的代码 (glimpse, summary, skim)。
    • 包含你绘制关键变量分布和变量关系图的 ggplot2 代码。
  4. 一份记录了你 EDA 主要发现的笔记或代码注释。

遇到困难怎么办？

如果在假期中遇到难以解决的问题 (找不到数据、R 代码报错、不确定如何解释图表等)，请记录下问题，可以在假期后向老师、助教或同学请教。

假期后工作坊：从探索到分析

恭喜你完成了五一假期的项目启动初步任务！希望你已经成功迈出了项目的第一步，拥有了初步的选题、数据和一些 EDA 的发现。

接下来的时间到第十三周课程开始前，你可以利用这段时间进行回顾和准备，更好地迎接后续更深入的分析、AI 辅助以及可重复报告的学习。

回顾与思考 (为工作坊和后续学习做准备)：

  • 回顾假期成果： 看看你是否已经基本完成了指南中”五一假期建议任务”列出的三点要求。
    • 你的研究问题是否足够清晰和具体了？
    • 数据是否已成功导入，并且你对它的结构和内容有了初步了解？
    • 你是否通过 EDA 发现了一些有趣或异常的现象？将这些发现记录下来。
  • 梳理遇到的问题： 在选题、找数据、导入数据或初步 EDA 过程中，你遇到了哪些具体的困难或技术问题？例如：
    • 不确定选题的数据能否支持？
    • 找不到合适的数据集？
    • 数据导入时出现乱码或格式错误？
    • 不理解某些变量的含义？
    • 某种图表不知道如何绘制？
    • 绘制出的图表难以解读？
    • 数据中有很多缺失值不知道怎么办？ 将这些问题记录下来，在课堂上提问，或准备下周用 AI 工具尝试解决。
  • 思考下一步分析方向： 基于你的研究问题和 EDA 的初步发现，简单思考一下接下来可能需要进行哪些分析。例如：
    • 如果你的因变量是分类的，可能会用到 Logistic 回归。
    • 如果想比较几组的平均值，可能会用到 t 检验或 ANOVA。
    • 如果想预测一个连续变量，可能会用到线性回归。
    • 你可能还需要对数据进行清洗或转换（如处理缺失值、异常值，或进行变量变换）。

为第十三周课程做准备：

第十三周我们将学习如何利用 AI 工具辅助数据分析过程，以及如何创建可重复报告 (使用 Quarto)。为了充分利用课堂时间，你可以：

  • 准备 AI 提问素材： 从你梳理的问题中，挑选 2-3 个具体的、可以用技术手段解决的问题，准备在下周课堂上或课后尝试用 AI 辅助解决。
  • 初步了解 Quarto： 如果有时间，可以简单浏览一下 Quarto 的官方网站 (https://quarto.org/) 或相关介绍，了解它如何将代码、结果和文字结合起来。思考如何用 Quarto 来组织你已有的项目代码和 EDA 发现。

项目时间表 (提醒)：

  • 第 13 周： 学习 AI 辅助分析技术和可重复报告制作；基于 EDA 结果，开始清理数据并构建初步统计模型。
  • 第 14 周： 深入模型评估与诊断；解释结果并进行讨论；开始撰写报告草稿。
  • 第 15-16 周： 完善分析并迭代模型；完成最终报告撰写和项目展示准备。
