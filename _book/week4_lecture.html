第四周：眼见为实：ggplot2 可视化探索与推断初步

1. 为何需要数据可视化？

我们已经学会了如何整理数据和计算描述性统计量，但数字往往难以直观地揭示数据背后的模式、趋势和异常。数据可视化通过图形将数据呈现出来，帮助我们：

  • 探索性数据分析 (Exploratory Data Analysis, EDA): 快速理解数据分布、变量间关系、发现异常点。
  • 沟通发现: 清晰有效地向他人展示数据洞见。
  • 模型诊断: (后续课程) 检查统计模型的假设是否成立。
本周目标
  • 理解数据可视化的基本原则。
  • 掌握 ggplot2 的核心语法和图层叠加思想。
  • 能够绘制常用的单变量和双变量图形。
  • 对图形进行基本的定制（标签、主题）。
  • 初步理解推断统计的核心思想：从样本推断总体。
  • 理解点估计和区间估计的概念。
  • 掌握置信区间的概念和正确解释。

2. 数据可视化原则

有效的可视化不仅仅是绘制图形，更要遵循一定的原则，确保图形清晰、准确、不误导。

  • 清晰性: 图形元素（点、线、条）易于区分，标签、坐标轴清晰明了。
  • 准确性: 图形真实反映数据特征，避免扭曲（如不恰当的坐标轴范围）。
  • 简洁性: 避免不必要的图形元素（“图表垃圾”），突出核心信息。
  • 目的性: 图形应服务于特定的分析或沟通目的。选择合适的图形类型。

3. ggplot2：图形的语法

ggplot2 是 tidyverse 中的旗舰级可视化包，它基于“图形语法”(Grammar of Graphics) 的理念，提供了一种强大而灵活的方式来创建各种复杂的图形。

  • 核心理念：图层叠加 (Layers) 一个 ggplot2 图形由若干图层叠加而成，每个图层包含：

    1. 数据 (Data): 要可视化的数据框或 Tibble。
    2. 映射 (Aesthetic Mappings, aes()): 将数据中的变量映射到图形的视觉属性（如 x 轴位置、y 轴位置、颜色、形状、大小等）。
    3. 几何对象 (Geometric Objects, geom_...): 定义了使用哪种图形来表示数据（如点、线、条形、箱线等）。
    4. 统计变换 (Statistical Transformations, stat_...): 对数据进行统计汇总（如计算频数、密度、均值等），geom 通常有默认的 stat。
    5. 位置调整 (Position Adjustments, position_...): 处理图形元素重叠问题（如堆叠、并列）。
    6. 坐标系 (Coordinate System, coord_...): 控制坐标轴（如笛卡尔坐标、极坐标）。
    7. 分面 (Faceting, facet_...): 将数据按某个分类变量分割成多个子图。

  • 基本语法模板:

    ggplot(data = <DATA_FRAME>) +
  geom_xxx(mapping = aes(<MAPPINGS>)) +       # 其他图层、设定...

  • 示例数据准备: 我们将使用 ggplot2 内置的 mpg 数据集（关于不同汽车型号燃油效率等信息）。

4. 常用单变量图形

用于探索单个变量的分布特征。

  • 直方图 (Histogram): geom_histogram()

    • 展示连续变量的分布情况。将变量范围分成若干区间 (bins)，计算落在每个区间的观测数量。
    • aes(): 通常只需要映射 x 轴。
    # 探索发动机排量 (displ) 的分布
ggplot(data = mpg) +
  geom_histogram(mapping = aes(x = displ), binwidth = 0.5, fill = "skyblue", color = "black")
      # binwidth 控制区间的宽度，可以调整以观察不同粒度的分布
  # fill 设置填充色, color 设置边框色

  • 密度图 (Density Plot): geom_density()

    • 直方图的平滑版本，更清晰地展示分布形状。
    • aes(): 通常只需要映射 x 轴。
    # 探索发动机排量 (displ) 的分布
ggplot(data = mpg) +
  geom_density(mapping = aes(x = displ), fill = "lightgreen", alpha = 0.7)
      # alpha 控制填充透明度

  • 箱线图 (Boxplot): geom_boxplot()

    • 展示连续变量的分布概要（中位数、四分位数、异常值）。
    • aes(): 通常映射 y 轴（或 x 轴，如果想画水平箱线图）。可以映射 x 轴为分类变量来分组比较。
    # 探索每加仑高速公路行驶英里数 (hwy) 的分布
ggplot(data = mpg) +
  geom_boxplot(mapping = aes(y = hwy), fill = "orange")
    # 查看不同驱动方式 (drv: f=前驱, r=后驱, 4=四驱) 的 hwy 分布
ggplot(data = mpg) +
  geom_boxplot(mapping = aes(x = drv, y = hwy, fill = drv)) # 按 drv 分组并用颜色区分
    • 箱体：覆盖 IQR (Q1 到 Q3)。
    • 箱内粗线：中位数 (Q2)。
    • 触须 (Whiskers)：通常延伸到距离箱体 1.5 倍 IQR 范围内的最远点。
    • 点：超出触须范围的潜在异常值。

5. 常用双变量图形

用于探索两个变量之间的关系。

  • 散点图 (Scatter Plot): geom_point()

    • 展示两个连续变量之间的关系。
    • aes(): 映射 x 和 y 轴。可以映射其他属性（如 color, shape, size）到第三个变量。
    # 探索发动机排量 (displ) 与高速公路里程 (hwy) 的关系
ggplot(data = mpg) +
  geom_point(mapping = aes(x = displ, y = hwy))
    # 将颜色映射到车辆类别 (class)
ggplot(data = mpg) +
  geom_point(mapping = aes(x = displ, y = hwy, color = class))
    # 将点的大小映射到气缸数 (cyl)
ggplot(data = mpg) +
  geom_point(mapping = aes(x = displ, y = hwy, size = cyl, color = class), alpha = 0.6)

  • 分组箱线图/小提琴图 (Grouped Boxplot/Violin Plot)

    • 比较一个连续变量在不同分类变量组别下的分布。
    • geom_boxplot() 或 geom_violin() (小提琴图结合了箱线图和密度图)。
    • aes(): 映射 x 到分类变量，y 到连续变量。
    # 比较不同车辆类别 (class) 的高速公路里程 (hwy) 分布 (箱线图)
ggplot(data = mpg) +
  geom_boxplot(mapping = aes(x = class, y = hwy, fill = class)) +
  coord_flip() # 翻转坐标轴，让类别标签更易读
    # 比较不同车辆类别 (class) 的高速公路里程 (hwy) 分布 (小提琴图)
ggplot(data = mpg) +
  geom_violin(mapping = aes(x = class, y = hwy, fill = class)) +
  geom_boxplot(mapping = aes(x = class, y = hwy), width = 0.1, fill = "white", alpha = 0.5) + # 叠加箱线图
  coord_flip()

  • 条形图 (Bar Plot): geom_bar() / geom_col()

    • geom_bar(): 统计分类变量的频数。aes() 只需要映射 x。
    • geom_col(): 直接绘制已计算好的高度值。aes() 需要映射 x 和 y。
    # 统计不同车辆类别 (class) 的数量
ggplot(data = mpg) +
  geom_bar(mapping = aes(x = class, fill = class)) +
  coord_flip()
    # 统计不同类型车辆的平均油耗 (hwy)
vehicle_summary <- mpg %>%
  group_by(class) %>%
  summarise(Avg_hwy = mean(hwy))

# 绘制每个车辆类别的平均油耗 (hwy)
ggplot(data = vehicle_summary) +
  geom_col(mapping = aes(x = class, y = Avg_hwy, fill = class))

6. 图形定制初步

ggplot2 提供了丰富的定制选项。

  • 标签 (Labels): labs()

    • 修改标题、副标题、坐标轴标签、图例标题等。
    p <- ggplot(data = mpg) +
  geom_point(mapping = aes(x = displ, y = hwy, color = class))

p + labs(
  title = "发动机排量与高速公路里程关系",
  subtitle = "数据来源：mpg 数据集",
  x = "发动机排量 (升)",
  y = "高速公路里程 (英里/加仑)",
  color = "车辆类别",
  caption = "图形由 ggplot2 生成"
)

  • 主题 (Themes): theme_...()

    • 快速改变图形的整体外观（背景、网格线、字体等）。
    • 内置主题：theme_bw(), theme_minimal(), theme_classic(), theme_light(), theme_dark() 等。
    • theme() 函数可以进行更细致的调整。
    p + labs(
  title = "发动机排量与高速公路里程关系",
  x = "发动机排量 (升)",
  y = "高速公路里程 (英里/加仑)",
  color = "车辆类别"
) +
theme_minimal() # 应用 minimal 主题

7. 推断统计思想初步

到目前为止，我们主要关注描述统计（描述已有数据的特征）和探索性分析（发现数据中的模式）。推断统计则更进一步，试图从样本 (Sample) 数据推断总体 (Population) 的特征。

  • 总体 vs 样本:
    • 总体: 我们感兴趣的所有个体或对象的集合（例如，所有中国大学生的身高）。总体通常太大而无法完全测量。
    • 样本: 从总体中抽取的一部分个体或对象的集合（例如，随机抽取 1000 名中国大学生的身高）。我们通过分析样本来了解总体。
  • 抽样分布 (Sampling Distribution):
    • 想象一下，我们反复从同一个总体中抽取多个大小相同的样本，并计算每个样本的某个统计量（如样本均值 \(\bar{x}\)）。
    • 这些样本统计量的分布就称为该统计量的抽样分布。
    • 抽样分布描述了样本统计量本身的不确定性或变异性。
  • 中心极限定理 (Central Limit Theorem, CLT) 直观理解:
    • CLT 是统计学中最重要的定理之一。
    • 核心思想: 无论总体本身的分布形状如何（只要不是太极端），当样本量 \(n\) 足够大时，样本均值 \(\bar{x}\) 的抽样分布近似服从正态分布。
    • 这个正态分布的均值等于总体均值 \(\mu\)，标准差（称为标准误 Standard Error, SE）等于总体标准差 \(\sigma\) 除以 \(\sqrt{n}\) (\(SE = \frac{\sigma}{\sqrt{n}}\))。
    • 意义: 使得我们可以利用正态分布的性质来对总体均值进行推断，即使我们不知道总体的具体分布。

CLT 是许多统计推断方法（如 t 检验、置信区间）的理论基础。它告诉我们，即使原始数据不是正态的，样本均值在多次抽样下也会趋向于正态分布，这极大地简化了推断过程。

8. 参数估计：点估计 vs 区间估计

我们通常希望估计总体的某个参数（如总体均值 \(\mu\)、总体比例 \(p\)）。

  • 点估计 (Point Estimate): 用样本统计量直接作为总体参数的估计值。
    • 例如，用样本均值 \(\bar{x}\) 估计总体均值 \(\mu\)。
    • 用样本比例 \(\hat{p}\) 估计总体比例 \(p\)。
    • 优点: 简单直观。
    • 缺点: 几乎不可能精确等于总体参数，且没有提供估计的不确定性信息。由于抽样误差，每次抽样的点估计值都会有所不同。
  • 区间估计 (Interval Estimate): 给出一个包含总体参数真实值的可能范围，并附带一定的置信水平 (Confidence Level)。
    • 最常见的区间估计是置信区间 (Confidence Interval, CI)。
    • 优点: 提供了估计的不确定性度量。
    • 缺点: 理解和解释比点估计复杂。

9. 置信区间 (Confidence Interval)

  • 概念: 基于样本数据构造的一个区间，我们有一定程度的信心（由置信水平决定）认为该区间包含了未知的总体参数。
  • 置信水平 (Confidence Level): 通常表示为 \((1-\alpha) \times 100\%\)，常用 95%。
    • 95% 置信区间的含义 (重要！): 如果我们反复从总体中抽取无数个相同大小的样本，并为每个样本构造一个 95% 置信区间，那么大约有 95% 的这些区间会包含真实的总体参数。
    • 错误理解: 不能说“总体参数有 95% 的概率落在这个具体的区间内”。总体参数是一个固定的值，它要么在区间内，要么不在。置信水平描述的是构造区间这个方法的长期可靠性。
  • 置信区间的构成 (以总体均值 \(\mu\) 为例，假设 \(\sigma\) 已知或 \(n\) 很大):
    • 置信区间 = 点估计 \(\pm\) 边际误差 (Margin of Error, MOE)
    • \(CI = \bar{x} \pm Z_{\alpha/2} \times SE = \bar{x} \pm Z_{\alpha/2} \times \frac{\sigma}{\sqrt{n}}\)
    • \(Z_{\alpha/2}\) 是标准正态分布上侧 \(\alpha/2\) 分位数（例如，95% 置信水平对应 \(\alpha=0.05\), \(Z_{0.025} \approx 1.96\)）。
    • 边际误差取决于：
      • 置信水平: 水平越高，区间越宽。
      • 数据变异性 (\(\sigma\)): 变异性越大，区间越宽。
      • 样本量 (\(n\)): 样本量越大，区间越窄（估计越精确）。
  • 解释示例: 假设我们计算出某产品用户满意度的 95% 置信区间为 [8.2, 8.8]。
    • 正确解释: 我们有 95% 的信心认为，该产品用户的真实平均满意度介于 8.2 和 8.8 之间。或者说，如果我们重复抽样并构造区间，预计 95% 的区间会包含真实的平均满意度。
    • 错误解释:
      • “用户的满意度有 95% 的概率在 8.2 到 8.8 之间。” (概率是针对区间的构造方法，不是针对具体区间或参数)
      • “95% 的用户满意度在 8.2 到 8.8 之间。” (置信区间是关于总体均值，不是关于个体数据)
